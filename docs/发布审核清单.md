## 发布前必须确认

- 微信公众平台主体信息已完成。
- 小程序名称、头像、简介已设置。
- 服务类目已选择，并且与「课时管理工具」用途一致。
- 隐私保护指引已在微信公众平台填写。
- 体验版已经用管理员、老师、家长三个角色测试通过。
- 云函数 `api` 已经上传并部署，选择「云端安装依赖」。
- 云数据库集合存在并可正常读写。
- 当前云环境中的测试数据已经处理好：发布后不要再随意清空当前环境。

## 服务类目建议

当前小程序功能是课时管理和记录工具，不提供在线课程内容、不收款、不招生、不发布培训广告。

在微信公众平台选择服务类目时，优先选择平台可选类目中最接近「工具 / 效率 / 教育管理 / 企业管理」的类目。最终以微信公众平台后台实际可选类目和审核提示为准。

如果后续增加在线课程、招生推广、付费购买、直播、音视频课程等能力，可能需要重新确认教育相关资质和服务类目。

## 微信开发者工具提交步骤

1. 使用新 AppID 打开本项目。
2. 确认 `project.config.json` 中 `appid` 是 `wxcf57d447d8205deb`。
3. 确认 `cloudfunctionRoot` 是 `cloudfunctions/`。
4. 右键 `cloudfunctions/api`，选择「上传并部署：云端安装依赖」。
5. 点击「编译」，确认无报错。
6. 点击「预览」，用手机完成管理员、老师、家长测试。
7. 点击「上传」，填写版本号 `0.1.0` 和版本描述。
8. 到微信公众平台后台提交审核。

## 当前版本的已知边界

- 发布版暂时继续使用开发环境 `cloud1-d8gtkgh0n2f7c8c5f`，测试数据和正式数据在同一环境中。
- 当前不实现订阅消息通知。
- 当前不实现数据导出。
- 当前不提供机构切换，一个微信账号通过创建或绑定进入一个机构。
- 当前家长端只读，只能查看已绑定的学员-课程数据。
