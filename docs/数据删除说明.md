# 课时小伙伴数据删除说明

本说明用于微信小程序发布审核和后续运营处理。当前版本支持部分业务数据在小程序内删除，完整机构数据删除需要由运营方在云开发后台处理。

## 用户如何提交删除请求

用户可以在小程序中提交：

1. 打开「我的」页面。
2. 点击「意见反馈」。
3. 在反馈内容中写明删除请求，例如：

```text
请删除我的家长账号绑定数据。
请删除某某学员的数据。
请删除某机构的全部测试数据。
```

4. 填写可联系的微信号、手机号或邮箱。
5. 提交后等待管理员或运营方处理。

## 管理员可在小程序内处理的数据

管理员可以在小程序内处理以下数据：

- 删除课程。
- 删除学员。
- 删除或停用成员绑定。
- 撤销打卡。
- 撤销充值。

为避免误删误改后无法追溯，当前版本会保留课时记录和操作日志。撤销操作不会简单抹掉历史，而是追加撤销记录。

## 家长账号解绑

家长请求解绑时，建议处理范围为：

- 删除或停用该家长对应的成员绑定。
- 删除该家长提交的意见反馈联系方式。
- 保留机构侧必要的学员课时记录，除非机构负责人确认删除。

原因：学员课时记录属于机构经营记录，可能同时涉及老师、管理员和家长三方确认。

## 机构全部数据删除

如机构负责人要求删除整个机构数据，需要在云开发数据库中按机构 ID 删除或清理相关集合数据。

当前主要集合包括：

| 集合 | 内容 |
| --- | --- |
| `organizations` | 机构信息 |
| `users` | 微信用户基础身份 |
| `memberships` | 用户与机构、角色、绑定码关系 |
| `courses` | 课程信息 |
| `students` | 学员-课程信息 |
| `lesson_records` | 课时充值、打卡、撤销记录 |
| `operation_logs` | 操作日志 |
| `feedbacks` | 意见反馈 |

处理前应先确认机构 ID，避免误删其他机构数据。

## 当前发布版本的限制

- 当前没有独立的数据导出功能。
- 当前没有一键删除整个机构的后台页面。
- 当前使用同一个云开发环境发布，测试数据和正式数据共用环境，发布后不要随意执行清空数据操作。
